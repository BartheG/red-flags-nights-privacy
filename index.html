Privacy Policy – Red Flags Nights

Last updated: January 6, 2026

Data Controller

Publisher of the Red Flags Nights application
Contact: guillaume.barthe.pro@gmail.com

App Description

Red Flags Nights is a local party game (2–6 players) with no user accounts and no cloud features.

Data Processed

  • User-provided data (local only): player first names or nicknames, stored exclusively on the device.
  • Game content: card texts (“flags”) are embedded directly in the application code and are not personal data.
  • Advertising data (automatic): device identifiers and technical information collected by the Google Mobile Ads (AdMob) SDK.

Purposes

  • Local game functionality.
  • Display of interstitial ads, aggregated ad performance measurement, and fraud prevention.

Legal Basis

  • Performance of a contract (app functionality).
  • Consent (personalized advertising via CMP/UMP).
  • Legitimate interest (non-personalized ads, security and fraud prevention).

Data Sharing

Data sharing is limited to Google / AdMob for advertising services.
Google Privacy Policy: https://policies.google.com/privacy

Consent and Advertising

  • Consent is managed via CMP/UMP for the EU, EEA, UK, and Switzerland.
  • Non-personalized ads are served if consent for personalized advertising is not given.

What We Do Not Do

  • No user accounts and no cloud storage.
  • No SDKs other than Google Mobile Ads.
  • No proprietary analytics, no geolocation, no cross-app tracking, and no data selling.

Data Retention

  • Local data: stored on the device until deleted by the user or app uninstallation.
  • Advertising data: not stored by the publisher.

User Rights

  • Local data can be deleted by uninstalling the app or via device settings.
  • Advertising consent can be managed or withdrawn via the CMP.

Children

The app is not intended for children.

Stores

Compliant with App Store and Google Play requirements.
